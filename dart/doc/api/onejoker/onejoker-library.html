Classes

CardStack
A CardStack is the basic card collection type for the library, used to implement whole decks, player hands, discard piles, Texas Hold'em boards, active tricks, solitaire tableaux, etc. It is a simple LIFO stack of cards, with a subset of the sorts of methods of native arrays or stacks or queues in many languages, but tuned with cards and simulations in mind. The generic CardStack should be used sparingly: its enclosing classes, Hand and Deck, are to be preferred for most uses as they have more error checking and specialized shortcuts.

OjRandom
This file contains utility functions that are used in multiple places in the library. Things like random number generation, shuffling, and sorting are present in the language already, but these versions are specifically tuned and optimized for playing cards and games.

Enums

Card
Enum for cards and their basic methods. Numbers very important here, not just for compatibility but specific algorithms will fail if changed (see README).
Rank
Enum for card ranks and their basic methods. As with suits, do not change the numbers.
Suit
Enum for card suits and their most basic methods. The actual numbers are important here: change them and you will break some tests and compatibility with the Rust code.
